# Arlene Colloquium — защищённый мессенджер

## Arlene Colloquium — это кроссплатформенное приложение для обмена сообщениями и файлами с асимметричным шифрованием (RSA 2048 бит) и безопасным WebSocket-соединением. Проект написан на Python с использованием фреймворка KivyMD и поддерживает работу на Android, Windows, Linux и macOS.
### Основные возможности

  - 🔐 Регистрация и авторизация по email с подтверждением через 6-значный код.

  - 🗝️ Асимметричное шифрование (RSA) всех сообщений и передаваемых файлов.

  - 👥 Личные чаты и групповые чаты (с возможностью установки аватара группы).

  - 📎 Отправка любых файлов с автоматической разбивкой на части и последующей сборкой.

  - 🖼️ Аватары пользователей и изображения для групповых чатов.
    
  - 📱 Адаптивный интерфейс для телефонов и планшетов (два режима отображения).

  - 🔁 Автоматическая синхронизация чатов и обновление списка диалогов.

  - 🚪 Экспорт/импорт приватного ключа для переноса аккаунта на другое устройство.

### Технологии

 1. Kivy / KivyMD — графический интерфейс.

 2. WebSocket (websocket-client) — транспорт.

 3. Cryptography (RSA, Fernet) — шифрование канала и конечных сообщений.

 4. PIL — работа с изображениями.

 5. Plyer — вызов системного диалога выбора файлов.

 6. Validators — проверка email.

 7. jnius (Android) — доступ к внешнему хранилищу на Android.

### Требования

  - Python 3.8 или выше

  - Зависимости из requirements.txt

### Установка и запуск (до официального релиза, в качестве бета-теста)

  - Клонируйте репозиторий
    ```bash

    git clone https://github.com/RKP16102052/arln_cllqm.git
    cd arln_cllqm
    ```

  - Создайте виртуальное окружение (рекомендуется)
    ```bash

    python -m venv venv
    source venv/bin/activate      # Linux/macOS
    venv\Scripts\activate         # Windows
    ```

    Установите зависимости
    ```bash

    pip install -r requirements.txt
    ```

    Запустите приложение
    ```bash

    python ui.py
    ```

### Особенности и ограничения

Для работы требуется собственный WebSocket-сервер, реализующий протокол, описанный в коде клиента. Серверная часть в репозиторий не включена.

Шифрование канала — Fernet (симметричное) с фиксированным ключом. Для производственной среды ключ необходимо генерировать и безопасно распространять.

Сообщения и файлы шифруются RSA-2048 (OAEP с SHA-256). Размер блока при шифровании — 180 байт (для сообщений) и 180 байт для файлов, что соответствует стандартным ограничениям RSA.

На Android требуется разрешение на чтение/запись внешнего хранилища (запрашивается автоматически через filechooser).

При первом входе после регистрации система запросит импорт приватного ключа (экспортированного с другого устройства). Это необходимо, так как приватный ключ никогда не передаётся по сети.
